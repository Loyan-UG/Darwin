## Cross-Cutting Rules

- Domain and Application stay provider-agnostic.
- Admin DTOs must not leak into public/member/mobile contracts.
- Provider secrets must not be exposed in WebAdmin, docs, logs, or tests.
- Production immutability, e-invoice compliance, provider delivery, and live payment readiness require real provider validation.
- Mobile is conditionally usable for implemented workflows, not fully store-launch ready until signed artifacts and device/provider smoke pass.
